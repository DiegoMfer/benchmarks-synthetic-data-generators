# LUBM Shape Extraction Experiment

This folder contains the scripts and results for generating LUBM synthetic data, converting it to a single Turtle file, and extracting ShEx and SHACL shapes.

## Prerequisites

- Python 3.6+
- Java 17+ (for LUBM generation)
- Python libraries: `rdflib`, `shexer`

## Steps to Reproduce

### 1. Generate LUBM Data
From the `LUBM` directory in the project root:
1. Clear previous output: `rm -rf output/*`
2. Run the generator (e.g., for 1 university):
   ```bash
   python3 execute_benchmark.py -u 1
   ```
   This creates `.owl` files in `LUBM/output`.

### 2. Convert to Turtle
From the `Auxiliar_folder` directory (this directory):
Run the conversion script to merge all `.owl` files into a single `lubm.ttl` file:
```bash
python3 convert_lubm_to_ttl.py
```

### 3. Extract Shapes
From the `Auxiliar_folder` directory (this directory):
Run the extraction script to infer ShEx and SHACL shapes from `lubm.ttl`:
```bash
python3 extract_shapes.py
```
This produces:
- `lubm.shex`: ShEx shapes.
- `lubm_shacl.ttl`: SHACL shapes.

## How it works

### Data Conversion
The `convert_lubm_to_ttl.py` script uses `rdflib` to parse the individual RDF/XML (`.owl`) files generated by LUBM and serializes them into a single coherent Turtle graph.

### Shape Extraction (`shexer`)
We use the `shexer` library (https://github.com/DaniFdezAlvarez/shexer) to automatically extract shapes.
- **Input**: `lubm.ttl`
- **Process**: `shexer` analyzes the instance data in the graph, identifying common patterns and typing for each class.
- **Output**:
  - **ShEx**: Extracted with a 90% acceptance threshold (properties appearing in <90% of instances are ignored).
  - **SHACL**: Extracted using the same logic but serialized to SHACL (Turtle format), saved as `.ttl` to allow for easy viewing and syntax highlighting.
