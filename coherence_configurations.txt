================================================================================
COHERENCE CONFIGURATIONS FOR RDF SYNTHETIC DATA GENERATORS
================================================================================

This document describes the configuration parameters used to achieve HIGH and
LOW coherence for each RDF data generator in the benchmark suite.

COHERENCE DEFINITION:
- HIGH COHERENCE (~1): Highly structured data where instances of a type have 
  values for almost all properties. Similar to relational databases (e.g., TPC-H).
- LOW COHERENCE (~0): Sparse/unstructured data where properties vary significantly
  between instances of the same type. Similar to LOD datasets (e.g., DBpedia).

Source Directory: PYGRAFT/

┌─────────────────────────────────────────────────────────────────────────────┐
│ HIGH COHERENCE                                                              │
├─────────────────────────────────────────────────────────────────────────────┤
│ mode: full                                                                  │
│ classes: 16              ← Fewer classes = more instances per class         │
│ relations: 12            ← Fewer relations = more uniform coverage          │
│ avg_instances: 500       ← More instances per class                         │

│ p_subclass: 0.05         ← Low subclass prob = flatter types                │

│                                                                             │

│ per class creates uniform, highly coherent data structure.                 │

│ LOW COHERENCE                                                               │
├─────────────────────────────────────────────────────────────────────────────┤
│ mode: full                                                                  │
│ classes: 64              ← Many classes = fewer instances per class         │
│ relations: 32            ← Many relations = varied property coverage        │
│ avg_instances: 100       ← Fewer instances per class                        │
│ max_depth: 5             ← Deep hierarchy = type variation                  │
│ p_subclass: 0.25         ← High subclass prob = complex hierarchy           │
│ runs: 3                                                                     │
│                                                                             │
│ Rationale: Deep hierarchy with many classes/relations and fewer instances  │
│ per class creates varied, sparse data with low coherence.                  │
